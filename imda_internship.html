I would like to thank my supervisor, Kwong Wai, for the valuable opportunity to intern at IMDA! I have learnt a lot under his mentorship and I'm truly grateful.

For my IMDA internship, I was tasked to create a solution that was able to detect data exfiltration characteristics and isolate possible data exfiltration connections. I decided to use data analytics to select data exfiltration features to aid in identifying possible data exfifltration behaviour.

To meet the objectives, I designed a two component outlier detection solution, consisting of preprocessing and visualisation. The preprocessing was scoped to consider both data exfiltration and port scanning behaviour during feature selection.

Preprocessing process:

  1. Data cleaning and feature engineering.
  2. Filtering of abnormal source IPs using the Isolation forest algorithm.
  3. Further filtering selected IPs from step 2 using specific variables.
  4. Categorsing the IPs as normal or outlier and saving it as a dataset.

The second component consists of the visualisation tool that visualises the curated data from the preprocessing step. This tool was built using Streamlit.

Use cases for the visualisation tool:

  • Users can parse data to visualise normal and outlier data points.
  • Different variables can be chosen to compare amongst individual source IPs and visualised in different plots.
  • Distribution of data points can be visualised to gain a better overall understanding.

These are some of my learning points during the internship:

  1. Big data preprocessing of upwards 50 million rows using PySpark
  2. Creating a new application from scratch using Streamlit
  3. Learning about data exfiltration behaviour and machine learning techniques for DE detection.
  4. Implementing and finetuning Isolation Forest algorithm
  5. Experimenting with high dimensionality density based algorithms and clustering algorithms like KModes and KPrototype.
